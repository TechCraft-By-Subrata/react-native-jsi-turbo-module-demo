# React Native JSI TurboModule Demo 🚀

Welcome to the **React Native JSI & TurboModule Sandbox**! This repository serves as a hands-on learning playground designed to teach you the modern mechanics of how cross-platform frameworks handle **zero-latency Inter-Process Communication (IPC)** using the JavaScript Interface (JSI).

In modern versions of React Native (0.76+), the old asynchronous JSON bridge is replaced entirely by JSI. Instead of serializing data into strings, JavaScript holds direct C++ memory references to native methods, unlocking **100% synchronous execution** and massive performance gains.

---

## 🛠️ Project Architecture

This application triggers a simple arithmetic addition task natively using direct C++ memory registers. Execution happens instantaneously on the calling thread without message queues or background serialization bottlenecks.

```text
[ JavaScript Engine ] ──(Direct Memory Pointer / JSI)──> [ C++ Layer ] ──> [ Native OS registers ]
                                                                                   │
                                                                           (Instant Addition)
                                                                                   │
[ JavaScript UI State ] <─────────────────(Direct Synchronous Return)──────────────┘
```

## 📂 Repository Structure

The core native files implemented in this project are located at:
* 🗂️ **Specs File:**
  * `specs/NativeCalculatorModule.ts` — The strongly typed TypeScript specification file that triggers the automated Codegen compiler engine.
* 🤖 **Android (Kotlin):** 
  * `android/app/src/main/java/com/jsiturbomoduledemo/CalculatorModule.kt` — Synchronous addition implementation mapping back to the code-generated specs.
  * `android/app/src/main/java/com/jsiturbomoduledemo/CalculatorPackage.kt` — Modern `TurboReactPackage` wrapper module.
  * `android/app/src/main/java/com/jsiturbomoduledemo/MainApplication.kt` — Package injection via modern Kotlin lists.
* 🍏 **iOS (Objective-C++):**
  * `ios/JsiTurboModuleDemo/CalculatorModule.h` — Module header file pointing to the autogenerated `<AppSpecs/AppSpecs.h>` framework interface.
  * `ios/JsiTurboModuleDemo/CalculatorModule.mm` — Core module logic utilizing Objective-C++ (`.mm`) for seamless runtime JSI memory sharing.

## ⚡ Quick Start Guide

### 1. Installation
Clone the repository and install the JavaScript dependencies:
```bash
git clone https://github.com
cd react-native-jsi-turbo-module-demo
npm install
```

### 2. Launching on Android
The New Architecture is turned on by default (`newArchEnabled=true` inside `gradle.properties`). Simply boot your emulator and run:
```bash
npx react-native run-android
```

### 3. Launching on iOS
Navigate into the native iOS directory, install the dynamically generated project specifications and dependency pods, and fire up your emulator:
```bash
cd ios
bundle exec pod install
cd ..
npx react-native run-ios
```
> **⚠️ Critical Xcode Note:** Make sure to link your custom `.h` and `.mm` native files inside your active Xcode project workspace using **"Reference files in place"** with your core application targets checked.
